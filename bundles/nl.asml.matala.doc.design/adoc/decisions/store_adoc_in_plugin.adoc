=== Store adoc in plugin

==== Decision
Store AsciiDoc documentation inside the plugin's own `adoc/` folder and add the adoc documentation to the binary artifacts.

==== Rationale
Keeping documentation with the plugin ensures documentation is versioned and distributed together with the code, avoids duplication, and makes it easy for consuming projects to reuse documentation from dependencies. 
For example, `nl.esi.comma.types` exposes its reference material in `adoc/`; downstream builds can include that file when assembling PDFs or site output.
